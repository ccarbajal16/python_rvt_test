Módulo RVT vis¶

Para cargar el archivo DEM en una matriz numpy y almacenar visualizaciones en GeoTIFF, usaremos el mdulo rvt.default (que se basa en la biblioteca gdal de python). Tambin podemos usar rasterio, gdal o cualquier otra biblioteca de Python.

In [1]:
import rvt.vis  # para generar las visualizaciones
import rvt.default  # para abrir y guardar nuestro rster
import numpy as np
import matplotlib.pyplot as plt  # para graficar nuestras visualizaciones.

En el directorio raster hay un archivo llamado hh_cut_final.tif, este ser nuestro DEM de prueba, desde el cual estaremos calculando visualizaciones. Defina la cadena con la ruta a este archivo (input_dem_path).

El mdulo default tiene la funcin get_raster_arr () que lee el raster segn la ruta indicada y devuelve un diccionario con las claves "array", "resolucin" y "no_data". La clave "array" es un numpy array del rster, la "resolucin" es una tupla del tamao de pxel donde el primer elemento es el tamao de pxel en la direccin x y el segundo es en la direccin y, "no_data" es el valor que representa noData en el raster (array).

In [2]:
dem_path = r'raster/hh_cut_final.tif' # Definiendo la ruta de nuestro DEM
dict_dem = rvt.default.get_raster_arr(dem_path) # Genera un diccionario a partir del DEM
dem_arr = dict_dem["array"]  # nuestra matriz numpy del DEM
dem_resolution = dict_dem["resolution"]
dem_res_x = dem_resolution[0]  # resolucin en la direccin X (0.5 para el ejemplo)
dem_res_y = dem_resolution[1]  # resolucin en la direccin Y (0.5 para el ejemplo)
dem_no_data = dict_dem["no_data"] # definiendo los valores considerados "NoData"

Funciones de visualización¶

1. Pendiente - Aspecto¶

Para calcular la pendiente y el aspecto, utilice la funcin rvt.vis.slope_aspect (). Los parmetros son: dem, resolution_x, resolution_y, output_units (puede ser: percent, degree, radianes), ve_factor, no_data, fill_no_data, keep_original_no_data. Nuestro DEM no tiene pxeles de datos (dem_no_data = None), por eso estableceremos fill_no_data y original_no_data en False. Salidas de funcin diccionario con las claves de slope y aspect. Cada clave contiene una numpy array.

In [3]:
# Generando el diccionario para generar las visualizacions de pendiente y aspecto
dict_slope_aspect = rvt.vis.slope_aspect(dem=dem_arr, resolution_x=dem_res_x,resolution_y=dem_res_y,
                                         output_units="degree", ve_factor=1, no_data=dem_no_data,
                                         fill_no_data=False, keep_original_no_data=False)

slope_arr = dict_slope_aspect['slope']
aspect_arr = dict_slope_aspect['aspect']

# Para visualizar la pendiente
plt.imshow(slope_arr, cmap='gray')

Para guardar la visualizacin, use rvt.default.save_raster (). Esta funcin toma los parmetros src_raster_path (ruta dem para copiar geodatos), out_raster_path (ruta de visualizacin), out_raster_arr (matriz numrica de visualizacin), no_data (cmo no se almacenan datos, todas las visualizaciones no_data se almacenan como np.nan), e_type (GDALDataType, por ejemplo, 6 es para float32 y 1 es para uint8).

2. Hillshade¶

Para calcular el sombreado, use la funcin rvt.vis.hillshade (). Los parmetros son: dem, ' resolution_x, resolution_y, sun_azimuth, sun_elevation, ve_factor, no_data, fill_no_data, keep_original_no_data. La funcin genera un numpy array de sombreado.

In [7]:
# Generar una visualizacin 'Hillshade'

3. Sombreado multidireccional¶

A diferencia de un sombreado predeterminado, el sombreado multidireccional presenta una vista incomparable de las montaas, mesetas, valles y caones del mundo mediante el uso de un algoritmo que calcula el sombreado desde seis direcciones diferentes.

Para calcular el sombreado de mltiples direcciones, use la funcin rvt.vis.multi_hillshade (). Los parmetros son: dem, resolution_x, resolution_y, nr_directions, sun_elevation, ve_factor, no_data, fill_no_data, keep_original_no_data. La funcin produce un numpy array 3D (donde la primera dimensin representa cada direccin (nr_directions), por ejemplo, arr[0] es la primera direccin).

In [9]:
# Generar una visualizacin de sombreado multidireccional
nr_directions = 16  # nmero de direcciones que corresponde al nmero de ngulos de azimut solar (nmero de bandas)
sun_elevation = 45  # ngulo vertical solar (sobre el horizonte) en grados.
multi_hillshade_arr = rvt.vis.multi_hillshade(dem=dem_arr, resolution_x=dem_res_x, resolution_y=dem_res_y, 
                                              nr_directions=nr_directions, sun_elevation=sun_elevation,ve_factor=1,                                              no_data=dem_no_data, fill_no_data=False, keep_original_no_data=False)

# grafica la primera direccin donde el azimut solar = 22,5 (360/16 = 22,5)
plt.imshow(multi_hillshade_arr[0], cmap='gray')

4. Modelo de Relieve Local Simple (SLRM)¶

Los modelos de relieve locales mejoran la visibilidad de las caractersticas de la superficie a pequea escala al eliminar los accidentes geogrficos a gran escala del DEM.

Para calcular el modelo de relieve local simple, use la funcin rvt.vis.slrm (). Los parmetros son: dem, radius_cell, ve_factor, no_data, fill_no_data, keep_original_no_data. La Funcin devuelve una gran variedad de modelos de relieve locales simples.

In [11]:
# Generar una visualizacin de modelo de relieve local simple 
radius_cell = 15 # radio a considerar en pxeles (no en metros)
slrm_arr = rvt.vis.slrm(dem=dem_arr, radius_cell=radius_cell, ve_factor=1, 
                        no_data=dem_no_data, fill_no_data=False, keep_original_no_data=False)

plt.imshow(slrm_arr, cmap='flag')

5. Modelo de Relieve Multi-Escala (MSRM)¶

El Modelo de relieve multi-escala (MSRM - Multi-Scale Relief Model), es un nuevo algoritmo para la interpretacin visual de accidentes geogrficos utilizando DSM (Modelos de Superficie Digital).La importancia de este nuevo mtodo radica en su capacidad para extraer la morfologa del relieve de los DSM de alta y baja resolucin, independientemente de la forma o escala del relieve en estudio.

Para calcular el modelo de relieve de mltiples escalas, utilice la funcin rvt.vis.msrm (). Los parmetros son: dem, resolution, feature_min, feature_max, scaling_factor, ve_factor, no_data, fill_no_data, keep_original_no_data. La funcin devuelve una gran variedad de modelos de relieve de mltiples escalas.

In [14]:
# Generar una visualizacin de Modelo de Relieve Multi-Escala
feature_min = 1 # Tamao mnimo de la caracterstica o razgo que desea detectar en metros.
feature_max = 5 # Tamao mximo de la caracterstica o razgo que desea detectar en metros.

msrm_arr = rvt.vis.msrm(dem=dem_arr, resolution=dem_res_x, feature_min=feature_min, 
                        feature_max=feature_max, scaling_factor=scaling_factor, ve_factor=1, 
                        no_data=dem_no_data, fill_no_data=False, keep_original_no_data=False)

plt.imshow(msrm_arr, cmap='flag')
